# Severity Rubric

- **BLOCKER**: would prevent building a correct MVP or cause major rework (e.g., missing route map, contradictory MVP scope, unknown entitlement names).
- **HIGH**: likely to create drift or major bugs (e.g., UI screens not matching PRD, missing error-state patterns).
- **MEDIUM**: causes friction or inconsistent UX (e.g., inconsistent component variants, unclear copy rules).
- **LOW**: polish, clarity, or future-proofing improvements.
